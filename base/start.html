Vue 改变了什么 ​

1.开发者体验(更少的代码) ​

我们来看一个基础的例子，针对于DOM元素的数据操作。

原生语法

Vue语法

说明

可以看到Vue语法相比于原生语法更加简洁，更加易读，更加易于维护，作为开发者，我们不需要关心底层的DOM操作，只需要关注业务逻辑即可。 在代码书写上，只要声明一个响应式数据，并将数据放到模版中,Vue会自动帮我们建立关联关系处理好DOM的更新，无需我们手动操作DOM。

名词解释DOM

DOM（Document Object Model）即文档对象模型，是W3C组织推荐的处理可扩展置标语言的标准编程接口。它定义了处理网页内容的模型以及如何通过脚本来操纵网页的内容、结构和样式。

大白话讲：就是我们写的标签如:<div>,便于在js中操作映射的对象,通过操作DOM对象,我们可以对网页的元素进行各种操作,比如添加、删除、修改、移动等。

名词解释响应式

在Vue中，数据变化时，视图会自动更新，而实现这一功能的机制就是响应式，响应式数据的生成要通过vue的内置函数reactive、ref、computed等来实现， 这些函数会返回一个响应式对象,当数据发生变化时，Vue会自动更新视图,如果数据源是非响应式数据,即使放入模版中,也无法实现视图的更新,因为Vue无法建立关联关系来实现更新。

2.开发者体验(对TypeScrip的支持) ​

javascript语法

javascript
const sum = (a, b) => a + b;
sum(1, 2); // 3
sum('1', '2'); // '12'

typescript语法

typescript
const sum = (a: number, b: number) => a + b;
sum(1, 2); // 3
sum('1', '2'); // Error: Argument of type '"1"' is not assignable to parameter of type 'number'.
1
2
3

说明

可以看到javascript语法在类型检查上并不严格，函数的参数约束性几乎没有。而typescript语法则更加严格，在开发者的编码习惯上更加规范，更加有利于代码的维护和扩展,同时规范的类型检查也能更好地提供代码提示，提高开发者的开发效率。 而Vue3.0版本默认内置typescript支持，可以更加方便地进行类型检查。

当然如果你对typescript还不了解，请参考TypeScript 教程(阮一峰)。

模块化代码

javascript
//模块化,通过模块化可以将公共函数、变量、对象等抽离出来,实现块化管理,提高代码的可维护性和扩展性。
// a.js-单独的js文件
export const GlobalConfig = {
    API_URL: 'http://localhost:3000'
};
export const commonFn = ()=>{
    console.log('commonFn');
}
// b.js-使用模块化的js文件
import { commonFn } from './a.js';
commonFn();

// c.js-使用模块化的js文件
import { commonFn,GlobalConfig } from './a.js';
commonFn();
if(GlobalConfig.API_URL === 'http://localhost:3000'){
    console.log('API_URL is correct');
}

说明

Vue支持ES6模块化，可以更加方便地进行代码的组织和管理，更加有利于代码的维护和扩展。

名词解释ES6

ECMAScript是JavaScript的标准，ES6是ECMAScript的第六个版本，它新增了很多有用的功能，包括模块化、异步编程、类、Promise等。

名词解释模块化

模块化是指将一个大程序分解成小的、可管理的、可复用的模块，每个模块只负责完成一部分功能，这样可以提高代码的可维护性和扩展性。

当然如果你对ES6模块化还不了解，请参考ECMAScript6 教程(阮一峰)。

4.开发者体验(丰富的生态系统) ​

  • 官方文档中文支持完美,可以让开发者更加方便地阅读。Vue 官方文档中文版
  • Vue生态系统丰富，提供了大量的插件、工具、组件，可以帮助开发者解决开发中的各种问题。
  • 基于npm仓库的包管理仓库支持，可以轻松安装第三方插件，提高开发效率。

名词解释npm仓库

npm仓库是一个公共的JavaScript包管理仓库，可以帮助开发者发布、安装、搜索、共享JavaScript代码。 在npm仓库中，可以搜索到大量的第三方插件、工具、组件，可以帮助开发者解决开发中的各种问题。
npm 仓库地址

5.开发者体验(组件化开发) ​

  • Vue提供了组件化开发的能力，可以将复杂的页面分解成多个小组件，每个组件只负责完成一部分功能，可以更加方便地进行代码的维护和扩展。
  • 可以这么理解,在Vue中,我们可以把一个复杂的页面拆分成多个组件,每个组件只负责完成一部分功能,然后通过组合这些组件来实现复杂的页面或复用.
  • 在使用时,只需要把组件想象成一个封装好的函数,导入到需要的地方,传入相应的数据,就可以实现页面的渲染。

6.开发者体验(更好的团队协作) ​

Vue中每一个组件可以是一个.vue文件,开发之间可以完成各自的业务逻辑，互不干扰。 Vue项目基于Node.js开发Vite构建,可以无缝接入git版本管理工具,eslint,prettier等代码规范工具,更加方便地进行团队协作。

名词解释Vite

Vite是一个快速的web应用开发构建工具，它使用ESBuild打包代码，并使用Rollup进行模块化，它可以极大地提高开发者的开发效率。 如果你对Vite还不了解，请参考Vite 教程(vitejs.dev)(非底层开发人员,建议后续学习)。
