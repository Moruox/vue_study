vue
<!--child-component.vue-->
<template>
  <div>
    <h1>Hello, World!</h1>
  </div>
</template>

<!--parent-component.vue-->
<template>
  <div>
    <child-component />
  </div>
</template>
<script lang="ts" setup>
  import ChildComponent from './child-component.vue'
</script>

组件传值props ​

vue
<!--child-component.vue-->
<template>
  <div>
<!--模版中使用props的属性时,可以直接使用，不需要加上-->
    <h1>{{ message }}</h1>
  </div>
</template>
<script lang="ts" setup>
  //定义该组件可以接收的参数及类型默认值,可以变向理解为在为函数定义参数
  const props = withDefaults(
      defineProps<{
          message: string
      }>(),{
          message: '默认值-- Hello, World!'
      }
  )
</script>
<!--parent-component.vue-->
<template>
  <div>
   <!--将数据传递给子组件-->
    <child-component :message="message"/>
  </div>
</template>
<script lang="ts" setup>
  import {ref} from 'vue'
  import ChildComponent from './child-component.vue'
  //定义父组件的数据
  const message = ref('How are you?')
</script>

说明

  • 组件传值是指父组件向子组件传递数据。
  • 使用defineProps方法，定义子组件可以接收的参数及类型。
  • 使用withDefaults方法，设置默认值。
  • 在父组件的模板中，使用:[props成员名称]="数据"的语法，将数据传递给子组件。

组件传值事件交互emit(最佳实践推荐) ​

说明

  • 在普通标签中,我们可以使用@符合+ 事件名="事件处理函数"来绑定事件(事件的触发是通过js引擎监听交互触发的),如果在组件上使用该语法,将会监听该组件的事件,当然前提是该组件上有相应的事件处理函数,此时触发条件,将由开发者来实现。
vue
<!-- 子组件-->
<template>
    <!--4.绑定事件 接收参数-->
  <input :value="modelValue" @input="modelValueUpdateHandler" />
</template>
<script lang="ts" setup>
  //1.定义可以接收的参数
const props = withDefaults(defineProps<{
      modelValue: string
    }>(),{
      modelValue: ''
})  
  //2.定义可以触发的事件(定义后,父组件可以触发该事件)
const emits = defineEmits(['update:modelValue']) 
  //3.监听input内容变化,并触发update:modelValue事件,将数据传递给父组件
const modelValueUpdateHandler = (newVal: string) => {
  emits('update:modelValue',newVal)
}
</script>
<!-- 父组件-->
<template>
  <ChildComponent :modelValue="inputValue" @update:modelValue="modelValueUpdateHandler" />
</template>
<script lang="ts" setup>
  //1.引入子组件
import ChildComponent from './ChildComponent.vue' 
import {ref} from 'vue'
  //2.准备数据源  
const inputValue = ref('')
  //3.监听子组件的update:modelValue事件,并更新数据源
const modelValueUpdateHandler = (newVal: string) => {
  inputValue.value = newVal
}
</script>

说明

  • 在上述代码中,父组件引入子组件,并绑定了子组件的update:modelValue事件,并将数据源inputValue与子组件的modelValue绑定,当子组件触发update:modelValue事件时,父组件将会接收到更新的数据,并更新inputValue的数据源,实现了数据的双向绑定。
  • 这种开发模式,可以有效地解耦组件,使得代码更加模块化,更容易维护,但也会增加开发难度，过于繁琐,为此vue提供了指令v-model来简化该模式。
  • v-model指令想当默认为子组件传入了modelValue与update:modelValue事件,并将数据源与子组件绑定,当子组件触发update:modelValue事件时,父组件将会接收到更新的数据,并更新inputValue的数据源,实现了数据的双向绑定。
  • 同时这也是官方推荐的最佳实践,所以在实际开发中,建议优先使用v-model指令来实现数据的双向绑定。
vue
<!-- 子组件-->
<template>
    <input v-model="modelValue" />
</template>
<script lang="ts" setup>
  //1.定义可以接收的参数
const props = withDefaults(defineProps<{
      modelValue: string
    }>(),{
      modelValue: ''
})  
  //2.定义可以触发的事件(定义后,父组件可以触发该事件)
const emits = defineEmits(['update:modelValue']) 
</script>
<!-- 父组件-->
<template>
  <ChildComponent v-model="inputValue" />
</template>
<script lang="ts" setup>
  //1.引入子组件
import ChildComponent from './ChildComponent.vue' 
import {ref} from 'vue'
  //2.准备数据源  
const inputValue = ref('')
</script>

组件引用生命周期变化 ​

说明

  • 当组件嵌套时,子组件的生命周期将会在父组件的onCreated之后执行,父组件的生命周期将会在子组件的onMounted之后执行。
  • 父组件beforeCreate => 父组件created => 父组件beforeMount =>子组件beforeCreate => 子组件created => 子组件beforeMount =>子组件mounted => 父组件mounted
  • 在开发中,应当注意该生命周期的执行顺序,避免出现意外情况。
