新建项目 ✳️ ​

说明

Vue的项目基于Node.js，请确保您的PC上已安装了对应的Node.js环境。 推荐Node.js版本为v16.13.0及以上。

名词解释Node.js

  • 我们常见的js是运行在浏览器上的，而Node.js则是跑在服务器上的JavaScript
  • 意味着Node.js有着像Java的系统级API调用权限，前后端分离的本质是利用了Node.js的能力,在本地建立服务，用作前后端交互的桥梁,将开发环境分离,提高开发效率.
  • Node.js也有着自己的包管理工具npm，可以方便地安装第三方包，提高开发效率(可以理解为java的maven)。
  • 参考：Node.js 简介(非底层开发人员, 建议后续学习)

环境检测
请打开终端，输入以下命令检测Node.js和npm是否安装

bash
node -v
npm -v

1.切换npm仓库源 ​

当Node.js安装成功时,自动携带npm仓库管理工具npm,但是此时默认的仓库地址是指向国外的,为了更好地维护使用依赖库，请执行下方指令将地址指向国内镜像地址。

bash
#设置npm镜像源为淘宝镜像源
npm config set registry https://registry.npmmirror.com
1
2

2.全局安装pnpm作为包管理工具 ​

  • 虽然Node.js自带了包管理工具npm，而pnpm对比npm有着更好的性能,和磁盘空间的优化,便于更好地管理依赖。
  • 后续篇章中我们将使用pnpm作为包管理工具。
bash
npm install -g pnpm@8 # node.js版本小于17.xx-执行该指令
npm install -g pnpm # node.js版本大于等于17.xx-执行该指令
# -g 全局安装

3.创建项目 ​

bash
pnpm create vite

bash

常见问题 ​

bash
#提示如下错误信息
error when starting dev server:
ReferenceError: structuredClone is not defined
1
2
3

导致该问题的核心原因是在node.js在17.0.0版本之前，structuredClone方法是实验性的，并没有被实装,而Vite6.0采用了该方法，因此需要变更node.js版本->18.0.0,或变更项目中的package.json中的vite版本为5.3.6或更低版本后重新安装依赖。

json
{
  "name": "vite-project",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc -b && vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "vue": "^3.5.13"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^5.2.1",
    "@vue/tsconfig": "^0.7.0",
    "typescript": "~5.6.2",
    "vite": "^6.0.0", 
    "vite": "^5.3.6", 
   "vue-tsc": "^2.2.0"
  }

名词解释Vite

  • Vite是一个快速的前端开发和构建工具，它使用Rollup打包代码，并在开发过程中提供丰富的功能，如热模块替换(HMR)、TypeScript支持、CSS预处理等。
  • Vite的核心理念是零配置，它可以自动优化开发环境，如自动安装依赖、提供丰富的插件、提供开箱即用的模板、提供零配置的开发服务器等。
  • 参考：Vite 官方文档(非底层开发人员, 建议后续学习)
