新建项目 ​

说明

Vue的项目基于Node.js，请确保您的PC上已安装了对应的Node.js环境。 推荐Node.js版本为v16.13.0及以上。

名词解释Node.js

Node.js是一个基于Chrome V8引擎的JavaScript运行环境，它是一个事件驱动的JavaScript运行时，用于快速、可靠地编写服务器端的JavaScript代码。
大白话:我们常见的js是运行在浏览器上的，而Node.js则是跑在服务器上的JavaScript，可以用来写后端。
参考：Node.js 简介(非底层开发人员, 建议后续学习)

1.切换npm仓库源 ​

当Node.js安装成功时,会自动携带npm仓库管理工具npm但是此时，默认的仓库地址是指向国外的,为了更好地维护使用依赖库，请执行下方指令将地址指向国内镜像地址。

bash
npm config set registry https://registry.npmmirror.com
1

2.全局安装pnpm作为包管理工具(推荐) ​

虽然npm也能安装包，但pnpm有着更好的性能，可以更好地管理依赖。

bash
npm install -g pnpm
# -g 全局安装
1
2

3.创建项目 ​

bash
pnpm create vite
1

接下来我们按照上面的提示依次执行

bash
cd vite-project
# 进入项目目录(如果当前终端已经在项目目录下，则不需要cd)
pnpm install
# 安装依赖
pnpm run dev
# 启动项目

常见问题 ​

bash
#提示如下错误信息
error when starting dev server:
ReferenceError: structuredClone is not defined
1
2
3

导致该问题的核心原因是在node.js在17.0.0版本之前，structuredClone方法是实验性的，并没有被实装,而Vite6.0采用了该方法，因此需要变更node.js版本->18.0.0,或变更项目中的package.json中的vite版本为5.3.6或更低版本后重新安装依赖。

json
{
  "name": "vite-project",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc -b && vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "vue": "^3.5.13"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^5.2.1",
    "@vue/tsconfig": "^0.7.0",
    "typescript": "~5.6.2",
    "vite": "^6.0.0", 
    "vite": "^5.3.6", 
   "vue-tsc": "^2.2.0"
  }
}
