运行逻辑 ✳️ ​

还记得上一篇文章中我们执行过的这个指令吗？

bash
pnpm run dev
1

这个指令实际是执行了我们项目根目录下的package.json 文件中定义的 scripts 下的 dev 脚本。

我们可以打开 package.json 文件，看一下 scripts 部分：

json
{
  "name": "vite-project",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vue-tsc -b && vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "vue": "^3.5.13"
  },
  "devDependencies": {
    "@vitejs/plugin-vue": "^5.2.1",
    "@vue/tsconfig": "^0.7.0",
    "typescript": "~5.6.2",
    "vite": "^5.3.6",
    "vue-tsc": "^2.2.0"
  }
}

可以看到,最终执行了一个叫 vite的命令,而vite 命令会启动一个本地服务，监听文件变化，并且实时编译代码，并提供热更新功能。 当通过浏览器访问本地服务的时候,会将根目录下的 index.html作为静态资源返回给浏览器。

疑问

  • 如果只返回了 index.html 文件,那么Vue是怎么和页面关联上的?
  • Vue是在什么时机渲染到页面上的?

让我们关注下index.html 文件

html
<!doctype html>
<html lang="en">
<head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Vite + Vue + TS</title>
</head>
<body>
<div id="app"></div>
<script type="module" src="/src/main.ts"></script>
</body>
</html>

  • 可以看到页面中并没有太多的东西，只有一个id="app"的div标签，并且引入了一个main.ts文件。
  • 当浏览器解析到<script type="module" src="/src/main.ts"></script>这行代码时，会去本地服务加载main.ts文件。
  • main.ts文件在src目录下，打开这个文件看一下：
typescript
//引入Vue的包,导出Vue提供的将组件渲染到页面的函数
import { createApp } from 'vue'
//导入css样式文件
import './style.css'
//导入App组件
import App from './App.vue'
//将组件渲染到页面上，并挂载到id为app的div标签上
createApp(App).mount('#app')

说明

  • 核心代码是导入了一个App.vue文件，并使用createApp函数将其渲染到id为app的div标签上。
  • .vue文件将在后面的章节中详细介绍，这里只需要知道它是Vue组件文件。
  • 可以理解为，Vue的渲染函数,会解析App.vue文件中的内容,并将解析后的内容渲染到id为app的div标签上。

基础项目结构 ​

vite-project
├── package.json
├── pnpm-lock.yaml
├── src
│   ├── App.vue
│   ├── assets
│   ├── components
│   ├── main.ts
│   └── style.css
├── tsconfig.json
└── vite.config.ts

  • package.json：项目配置文件，定义了项目的依赖、命令等信息。
  • pnpm-lock.yaml：项目依赖锁定文件，用于保证项目依赖的一致性。
  • src：项目源码目录，包含了项目的主要逻辑代码。
    • App.vue：项目的根组件，用于渲染整个页面。
    • assets：静态资源目录，包含了项目的图片、字体等资源。
    • components：组件目录，包含了项目的公共组件。
    • main.ts：项目的入口文件，第一个加载的脚本文件。
    • style.css：样式文件，包含了项目的样式。
  • tsconfig.json：TypeScript配置文件，用于配置TypeScript的编译选项。
  • vite.config.ts：Vite配置文件，用于配置Vite的编译选项。

  • dist：项目编译后的输出目录。
  • node_modules：项目依赖目录。
  • public：静态资源目录，包含了项目的图片、字体等资源。
  • src：项目源码目录，包含了项目的主要逻辑代码。
    • api：接口目录，包含了项目的接口文件。
    • assets：静态资源目录，包含了项目的图片、字体等资源。
    • components：组件目录，包含了项目的公共组件。
    • views/page：页面目录，包含了项目的页面组件。
    • router：路由目录，包含了项目的路由配置。
    • styles：样式目录，包含了项目的样式文件。
    • utils：工具目录，包含了项目的工具函数。
    • App.vue：项目的根组件，用于渲染整个页面。
    • main.ts：项目的入口文件，第一个加载的脚本文件。
    • shims-vue.d.ts：TypeScript类型定义文件，用于解决TypeScript对Vue的类型定义缺失问题。
  • tests：测试目录，包含了项目的测试用例。
  • .env：环境变量配置文件，包含了项目的环境变量配置。
  • .env.development：开发环境变量配置文件，包含了项目的开发环境变量配置。
  • .env.production：生产环境变量配置文件，包含了项目的生产环境变量配置。
  • index.html：项目的入口HTML文件，用于配置项目的入口文件。
  • vite.config.ts：Vite配置文件，用于配置Vite的编译选项。
  • package.json：项目配置文件，定义了项目的依赖、命令等信息。
  • README.md：项目说明文档。
  • tsconfig.json：TypeScript配置文件，用于配置TypeScript的编译选项。
  • pnpm-lock.yaml：项目依赖锁定文件，用于保证项目依赖的一致性。
