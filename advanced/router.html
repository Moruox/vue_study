概念(Vue Router) ⭐️ ​

  • 按照目前的情况,我们两个页面需要导入App.vue文件,在App.vue文件中进行页面切换逻辑实现,这显如果随着项目体积的增多,这种方式是非常不合理的。
  • 我们需要一个更加灵活的方案,能够实现自动的页面切换,这时Vue官方的Vue Router就派上了用场
  • Vue Router的核心概念,将URL和组件进行映射,当URL发生变化时,Vue Router会自动匹配对应的组件进行渲染
  • 使用Vue Router的时候,你就会发现页面切换时,index.html中的<div id="app"></div>内部会自动替换成对应的组件，而无需刷新页面，这也是Vue Router的特性SPA应用的体现

名词解释SPA应用

  • Single Page Application单页应用
  • SPA的特点是只加载一次页面,然后通过路由切换不同的页面,实现了无刷新的效果
  • SPA的实现方式是通过HTML5 History API和Vue Router来实现的
  • HTML5 History API可以让我们在不刷新页面的情况下,通过URL的变化来切换页面
  • Vue Router可以帮助我们实现页面的切换,并自动匹配对应的组件进行渲染

安装 ⭐️ ​

bash
pnpm install vue-router

在src目录下创建router文件夹,然后在router文件夹下创建index.ts文件,作为配置文件使用

typescript
//导入createRouter和createWebHistory`
import { createWebHistory , createRouter } from 'vue-router'
//定义当前应用的路径和组件映射关系
const routes = [
    {
        //访问的路径地址
        path: '/',
        //当前页面的名字(需要跳转时可以当作key使用)
        name: 'Home',
        //当前页面展示的组件
        component: () => import('../page/todoList/index.vue')
    },
    {
        path: '/user',
        name: 'User',
        component: () => import('../page/user/index.vue')
    }
]
//创建路由实例
const router = createRouter({
    //配置路由模式
    history: createWebHistory(),
    //配置路由映射关系
    routes,
})
//导出路由实例
export default router

使用 ⭐️ ​

import App from './App.vue'
//导入路由实例
import router from './router'
import 'element-plus/dist/index.css'
const app = createApp(App)
//挂载路由实例到app实例上
app.use(router)

说明RouterView组件

  • RouterView组件是Vue Router自带的内置组件,用来指定当前渲染组件的位置。
  • 当路由发生变化时,RouterView组件会自动渲染对应的组件。
  • 他的变化渲染规则会参考我们前面的配置文件内的routes数组。

页面切换 ✳️ ​

对于上问题Vue Router提供了useRouter函数,通过useRouter方法可以获取到Vue Router实例完成页面的切换,请看具体使用

1.push方法(页面跳转) ​

  • Vue Router内置了一个页面跳转函数,可以通过指令的方式来实现页面的切换,使用方式如下
Vue
<template>
  <el-button @click="goHome">跳转到首页</el-button>
  <el-button @click="goUser">跳转到用户管理</el-button>
</template>
<script lang="ts" setup>
//导入useRouter方法  
import { useRouter } from 'vue-router'
//获取路由实例
const router = useRouter()
const goHome = () => {
  //跳转到首页
  router.push('/')
}
const goUser = () => {
  //跳转到用户管理
  router.push('/user')
}
</script>

push方法的详细介绍 ​

  • push方法可以用来跳转到不同的页面,并且可以携带参数
  • push方法支持两种调用方式,一种是传递一个字符串参数,另一种是传递一个对象参数
  • 传递字符串参数的形式如下
typescript
//将直接跳转到/user页面
router.push('/user')
1
2
  • 传递对象参数的形式如下
typescript
//将跳转到/user页面,并携带参数id=1
router.push({
  path: '/user',
  // name: 'User',(另一种跳转方式,不可和path同时使用,name的值需要和路由映射关系中的name一致）
  query: {
    id: 1
  }
})

2.replace方法(页面替换) ​

  • replace方法和push方法的用法基本一致,但是replace方法不会向history栈添加新记录,而是替换掉当前的history记录,因此不会导致页面的后退按钮失效
  • replace方法的调用方式如下
typescript
//将直接替换掉当前页面,跳转到/user页面
router.replace('/user')

页面跳转参数读取 ✳️ ​

  • Vue Router提供了useRoute函数,通过useRoute方法可以获取到当前路由的信息,包括query、name等信息,请看具体使用

1.useRoute方法 ​

  • useRoute方法可以用来获取当前路由的信息,包括query、name等信息
  • useRoute方法的调用方式如下
typescript
//导入useRoute方法
import { useRoute } from 'vue-router'
//获取路由实例
const route = useRoute()
console.log(route.query) //获取查询字符串参数(会自动解析成对象，如?id=1会被解析成{id:1})
console.log(route.name) //获取路由名称

结合前面的知识点,让我们小小的改造一下App.vue文件,实现页面的切换和参数的读取

vue
<template>
  <el-button :type="route.path === '/'? 'primary' : 'default'"  @click="router.push('/')">Go to todoList Page</el-button>
  <el-button :type="route.path === '/user'? 'primary' : 'default'" @click="router.push('/user')">Go to User Page</el-button>
  <el-button @click="router.back()" >back Page</el-button>
  <RouterView></RouterView>
</template>
<script lang="ts" setup>
import { useRouter , useRoute } from 'vue-router'
const router = useRouter()
const route = useRoute()
</script>

最终效果 ​

注意

  • 每一次的路由切换,都会导致当前组件被销毁，即触发生命周期函数onUnMounted调用
  • 当页面切换完成后,组件会重新生成,即触发onMounted生命周期函数调用
  • 更多细节请参考Vue Router官方文档
